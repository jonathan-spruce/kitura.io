WHAT IS ROUTING?

Routing determines how your Kitura server will respond to requests from a client on a given endpoint. An endpoint is the URI (the path) and the specific HTTP request method.

HTTP methods indicate a desired action to be performed on the given URI, and there’s many methods out there for this. The main ones are:

  • GET
  • POST
  • PUT
  • PATCH
  • DELETE

You can find a more exhaustive list with descriptions here

An example of registering an endpoint:

router.get(“/hello”) { … }

Where router is an instance of a Kitura router, get is the HTTP method to be used on the URI and “/hello” is the URI.

You can also register multiple endpoints on a Kitura server:

router.get(“/hello”) { … }
router.post(“/hello”) { … }
router.put(“/hello”) { … }
router.get(“/world”) { … }
router.post(“/world) { … }

Kitura offers two options for routing, Raw routing and Codable routing. Both of these options are provided with Kitura out of the box.
